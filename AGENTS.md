# AGENTS.md

本文件用于约束 AI 编程助手在当前工程中的工作方式。其作用范围为本文件所在目录及其所有子目录。

## 工程背景

- 当前工程是轮腿机器人控制工程，处于平衡验证阶段。
- 修改控制逻辑时，应优先保持现有调试链路可用，包括 WiFi 调试页面、运行时状态快照、左右轮速度控制接口和安全停止逻辑。
- 当前阶段优先采用便于实车验证和快速回退的控制方案，不直接跳到复杂控制架构。

## 工作规则

- 不需要进行 PlatformIO 编译，仅做源码检查、接口检查和静态一致性检查即可。
- 修改前先检查相关控制链路，避免只根据控制理论直接改代码。
- 不要回退或覆盖用户已有修改；如果遇到无关的已修改文件，保持原状。
- 平衡控制相关改动应尽量小步进行，保留原有 PD 调参路径，新增参数应有安全默认值。
- 涉及左右轮速度、编码器方向、输出方向时，必须明确说明符号约定，避免直接混用原始左右轮速度。
- Web 调试页面已有的调参和状态查看流程应尽量保留并扩展，避免把新功能隐藏到只能重新烧录才能调整的参数中。
- 注释和说明优先使用中文，变量名和代码风格保持项目现有习惯。

## 调试建议

- 平衡控制调参应先确认 IMU Pitch 方向、输出方向和左右轮速度方向，再逐步增加控制参数。
- 新增控制参数默认值应尽量不改变原有行为，例如轮速反馈系数 `Kv` 默认应为 `0.0`。
- 如果实车表现异常，应优先区分符号方向错误、速度环响应问题、保护逻辑触发和硬件执行问题。
