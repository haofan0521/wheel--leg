# 双核运行时架构说明

## 设计目标

本工程已经从单一 `setup()/loop()` 结构重构为基于 `FreeRTOS` 的双核任务框架，目标如下：

- 将实时控制链路与非实时服务链路分开
- 减少 Wi-Fi、网页和日志输出对 FOC 控制的打断
- 为后续编码器采样、IMU 读取、FOC 控制与遥测服务留出稳定扩展点

## 当前任务划分

### 控制任务

- 任务名：`control_task`
- 运行核心：`Core 1`
- 职责：
  - 初始化电驱模块
  - 初始化编码器模块
  - 初始化 IMU 模块
  - 初始化舵机模块
  - 发布控制任务状态快照
  - 预留后续 FOC 主循环接入位置

### 服务任务

- 任务名：`service_task`
- 运行核心：`Core 0`
- 职责：
  - 启动 Wi-Fi 服务
  - 运行网页调试接口
  - 发布服务任务状态快照
  - 预留后续遥控、遥测与参数调试接口

## 共享状态原则

- 控制任务与服务任务之间通过共享状态快照交互
- 网页调试页面只读取共享状态，不直接访问控制模块
- 后续新增状态字段时，优先加入共享状态结构体，而不是让网页直接调用底层驱动

## 下一步建议

1. 在控制任务中接入 `MT6835` 读取流程
2. 在控制任务中接入 `BMI088` 读取流程
3. 将 FOC 主循环固定到控制任务
4. 将遥控和参数调试接口放到服务任务
